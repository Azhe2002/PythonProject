# 2026-05-11 会话记录 — C_DEF 项目

## 项目概述

**C_DEF** = AI 生成人脸检测识别系统。四分支融合（物理噪声 + 频域残差 + RGB 语义 + 人脸光滑度），输入图像 → 输出 AI 生成概率。

## 今日产出

### 代码

- **`fusion_lr/config.json`** — 数据集路径 + 训练超参数（C 网格、5 折 CV）
- **`fusion_lr/extract_branch_scores.py`** — 对图像逐张调用 detector._branch_scores()，提取 4 维 [P, F, R, S] 分数，存为 .npy；支持 train/valid/test 多分片
- **`fusion_lr/train.py`** — 逻辑回归训练 + 交叉验证选 C + 手工权重基线对比，输出 AUC/F1/Acc/ECE + 原始系数 + 相对重要性（`|w_i|/sum(|w|)`，和=1）

- **`fusion_lr/result/data1.md`** — data1 实验结果
- **`fusion_lr/result/data2.md`** — data2 实验结果

### Bug 修复

- **Windows glob 重复**：`glob("*.jpg")` 和 `glob("*.JPG")` 在大小写不敏感文件系统返回相同文件，改用 `iterdir()` + `suffix.lower()` 去重
- **权重归一化错误**：原用 `w/sum(abs(w))` 致绝对值之和=1 但代数和≠1，改为相对重要性 `|w_i|/sum(|w|)` + 原始系数并列展示

## 方案一实验结果

### 数据概况

| | data1 | data2 |
|---|---|---|
| 来源 | Kaggle Real and Fake Face Detection | 140k Real and Fake Faces (FFHQ+StyleGAN2) |
| 训练集 | 2,041 (real=1081, fake=960) | 100,000 (real=50K, fake=50K) |
| 验证集 | 无 | 20,000 |
| 测试集 | 无（用 CV 代替） | 20,000 |

data3 未下载，cross_gen_test/ 弃用。

### 四分支分数 real/fake 差异

| 分支 | data1 diff | data2 diff |
|---|---|---|
| 物理噪声 | +0.0075 | +0.0093 |
| 频域残差 | -0.0020 | +0.0075 |
| RGB 语义 | +0.0025 | -0.0143 |
| 人脸光滑度 | -0.0154 | +0.0120 |

**所有差异均在 0.015 以内**，两类在四维特征空间中高度重叠。

### 性能对比

| 数据集 | 方法 | AUC | F1 | Acc | ECE |
|---|---|---|---|---|---|
| data1 | 手工基线 | 0.5036 | 0.0062 | 0.5311 | 0.1256 |
| data1 | 逻辑回归 | 0.5409 | 0.3087 | 0.5282 | 0.0085 |
| data2 | 手工基线 | 0.5561 | 0.0000 | 0.5000 | 0.1805 |
| data2 | 逻辑回归 | 0.5713 | 0.5706 | 0.5528 | 0.0018 |

逻辑回归在 data2 上将 AUC 从 0.556 → 0.571（+0.015），ECE 从 0.18 → 0.002（校准大幅改善），但**整体判别力仍然很弱**。

### 学习到的权重（data2）

| 分支 | 手工权重 | LR 原始系数 | LR 相对重要性 |
|---|---|---|---|
| 物理噪声 P | 0.460 | -1.970 | 0.241 |
| 频域残差 F | 0.320 | 0.146 | 0.018 |
| RGB 语义 R | 0.100 | 4.781 | 0.584 |
| 人脸光滑度 S | 0.120 | 1.291 | 0.158 |

- RGB 语义成为主导分支（相对重要性 58%），频域残差几乎被忽略（1.8%）
- 物理噪声原始系数为负，与该分支的理论预期矛盾
- 手工权重完全失效（F1=0，全部预测为同一类）

---

## 核心结论

**方案一（逻辑回归融合四分支权重）已验证完毕，上限为 AUC≈0.57。** 限制不在融合策略，而在四分支特征本身——当前 21 维手工特征 → 4 分支评分管道无法有效区分 FFHQ 真实人脸与 StyleGAN2 生成人脸。原因是 FFHQ 经过对齐/裁剪预处理，传感器噪声等物理特征已被削弱；StyleGAN2 的 artifact 模式与手工特征设计的假设不匹配。

**方案一不具备继续优化的价值，后续应从特征层面或模型层面升级。**
